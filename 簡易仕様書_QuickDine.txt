# 簡易仕様書 — QuickDine

### 作者
Vrowdice

### アプリ名
QuickDine

#### コンセプト
地図上で検索位置を選び、日本国内の飲食店をすぐ探せる。

#### こだわったポイント
- 全画面 Google マップを中心にした検索 UX（地図タップ・GPS・クイックピン）
- 検索条件を豊富に選択できる（検索半径・取得件数・ジャンル・駐車場・個室）
- 検索結果を地図マーカーとボトムシート一覧の両方で表示
- 結果シートの「おまかせ」ボタンでランダムに店舗を選び詳細画面へ遷移
- お気に入り・クイックピンのローカル保存、多言語対応（韓国語 / 日本語 / 英語）
- ホットペッパーグルメ Web サービスのクレジット表記を利用規約どおり実装

### 開発環境構築手順
1. プロジェクトを取得（Git クローンまたは zip 展開）
2. `flutter pub get` を実行
3. `assets/env.example` を `assets/env` にコピーし、API キーを設定
   - HOTPEPPER_API_KEY（リクルート Web サービスで取得）
   - GOOGLE_MAPS_API_KEY（Google Cloud Console で取得）
4. Android: `assets/env` の Maps キーは Gradle ビルド時に AndroidManifest へ自動注入
5. iOS: 起動時に `MapsKeyService` が env の Maps キーをネイティブへ渡す
6. `flutter run` で実機またはエミュレーター起動
7. HotPepper は日本国内データのみ → 海外からテストする場合は東京駅付近など日本の座標で検索

## アプリケーション機能

### 機能一覧
- レストラン検索：ホットペッパーグルメサーチ API を使用し、指定座標・検索半径（300m〜3000m）・取得件数（10〜100件）で周辺店舗を検索する
- 検索条件（任意追加）：ジャンル（8種）、駐車場あり（parking=1）、個室あり（private_room=1）
- 現在地取得：GPS（geolocator）で現在地を取得し、地図中心を移動する
- 地図検索：Google Maps 上でタップして検索中心を指定する
- クイックピン：よく使う地点に名前付きピンを保存し、ワンタップで移動する
- 検索結果表示：地図上に店舗マーカーを表示し、ボトムシートで一覧表示する（店舗名称 / アクセス / サムネイル画像）
- ランダム選択：結果シート左上のボタンで検索結果から1件をランダムに選び詳細画面を開く
- 店舗詳細表示：店名・写真・住所・営業時間・アクセス・ジャンル・キャッチコピー・平均予算を表示する
- 電話アプリ連携：店舗に電話番号がある場合、url_launcher で tel: スキームを起動する
- Web 予約ページ連携：HotPepper の店舗 URL を外部ブラウザで開く
- 地図連携（アプリ内）：詳細画面またはお気に入りから店舗位置を検索地図上に表示する
- お気に入り：店舗を端末内に保存・解除し、一覧から再表示できる
- 設定：デフォルト検索半径・取得件数・表示言語の変更、データ一括削除
- クレジット表示：API データ・画像利用画面にホットペッパー公式クレジットを表示する

### 画面一覧
- スプラッシュ画面（SplashScreen）：起動時ロゴ表示、初期化後に検索画面へ遷移
- 検索条件入力画面（SearchScreen）：GPS・検索半径・取得件数・ジャンル・設備フィルタを指定し、検索を実行する（課題の「検索条件入力画面」に相当）
- 検索結果一覧（SearchScreen 内ボトムシート）：検索結果の飲食店を一覧表示する。ドラッグで高さ調整、左上ランダム選択、右上閉じる（課題の「検索結果画面」に相当。独立画面ではなく地図連動の DraggableScrollableSheet）
- 店舗詳細画面（DetailScreen）：選択した店舗の詳細情報を表示する
- お気に入り画面（FavoritesScreen）：保存済み店舗の一覧
- 設定画面（SettingsScreen）：検索デフォルト値、言語、データ削除、アプリ情報

### 利用技術（フレームワークやAIなど）の選定理由
- **Flutter / Dart**：課題で許可されている開発言語のうち、習熟度が最も高く、Android・iOS を単一コードベースで提出できるため選択した。
- **Google Maps**：検索中心の視覚的な指定と結果マーカー表示に必須と判断した。
- **生成 AI の役割分担**
  - **Cursor**：Dart / Flutter のコード実装、リファクタリング、デバッグ支援に利用。生成コードは採用前に内容を確認し、理解した上で取り込んだ（課題補足事項に準拠）。
  - **Google Gemini（NotebookLM）**：機能企画、HotPepper API 調査、簡易仕様書の構成整理、README 文案のたたき台作成など、**設計・ドキュメント段階**で利用。実装コードの直接生成は Cursor 側で行った。

### アプリアイコン・ロゴ（Photoshop 自作）
- **制作ツール**：Adobe Photoshop（AI 生成画像ではなく、作者が直接デザイン・書き出し）
- **ストア用マスター**
  - `AppIcons/appstore.png` — App Store 用（1024×1024 相当）
  - `AppIcons/playstore.png` — Google Play 用（512×512 相当）
- **各サイズ一式**：`AppIcons/Assets.xcassets/`（iOS）、`AppIcons/android/mipmap-*/`（Android）
- **デザインコンセプト**
  - **地図ピン（位置情報）** のシルエットをベースに、その内側を **皿（食事・レストラン）** の円形で表現
  - 皿の上に **箸（和食・飲食）** を重ね、位置情報から飲食店を探すアプリであることを一目で伝える
  - 配色：クリーム色のピン本体、テラコッタ（橙）の皿、ダークグリーンの箸 — アプリ UI の primary / secondary トーンと統一
  - フラット・ミニマルなベクター調で、小サイズ（ホーム画面アイコン）でも判別しやすい構成

### デザイン面でこだわったポイント
- **アプリアイコン（Photoshop 自作）**：ピン＋皿＋箸で「位置 × 飲食店検索」を象徴（上記「アプリアイコン・ロゴ」参照）
- テラコotta（primary）+ セージグリーン（secondary）のブランドカラー、primary 色の AppBar — アイコン配色と統一
- 検索パネルを 1 枚のカードに統合（半径・件数 + 条件チップ + ジャンルチップ）
- 条件チップ（FilterChip・チェックマーク付き）とジャンルチップ（ChoiceChip）を区切り線で視覚分離
- 画面下部中央の Pill 型検索ボタンで片手操作を最適化
- ボトムシートと検索ボタンの位置連動（`ValueNotifier` + `SearchOverlayMetrics` でドラッグ時の地図パディングのみ更新）

### 技術面でアドバイスして欲しいポイント
- 検索結果のページネーション（API `start` パラメータ）と「もっと見る」の UX 設計
- 海外ユーザー向けに「日本データのみ」制約をどう案内するか
- HotPepper 検索 API では電話番号・URL が返らない店舗が多い — 詳細取得 API の併用可否
- 一覧リストの情報密度（ジャンル・アクセス・サムネイル）のさらなる見やすさ改善

### 実装メモ（エージェント向けドキュメント）

### 自己評価
課題の必須要件（GPS・検索半径・一覧3項目・詳細4項目・HotPepper API 利用）はすべて満たしている。一覧を独立画面ではなく地図連動ボトムシートにした点は UX 重視の判断だが、課題の「一覧表示」要件は機能として充足している。検索条件の追加（ジャンル・駐車場・個室）やお気に入り・電話連携など、最低限を超える実装も行った。ページングは 1 ページのみ（課題許容範囲）。今後はページネーションと詳細 API 連携で情報の充実度を上げたい。

## 課題要件との対応（補足）

| 課題要件 | 本アプリでの実装 |
|---------|----------------|
| GPS で現在地取得 | SearchScreen 起動時 + 現在地ボタン |
| 検索半径の指定 | 300m〜3000m（5段階） |
| その他検索条件（任意） | ジャンル、駐車場、個室、取得件数 |
| 一覧：店舗名称・アクセス・サムネイル | ShopListTile（ボトムシート） |

| 詳細：名称・住所・営業時間・画像 | DetailScreen |
| 詳細にふさわしい機能 | 電話、Web、予算、お気に入り、地図表示 |
| ランダム選択（追加機能） | 結果シート左上ボタン → 1件ランダム → DetailScreen |
